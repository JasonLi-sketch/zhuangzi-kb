# 庄子寓言事件标注方法论 (v1.0)

## 一、核心原则

### 1.1 以寓言为单位的事件提取

与《传习录》《孟子》的问答对话不同，庄子哲学的核心表达方式是**寓言**。每个寓言是一个独立的哲学事件，包含：
- **故事层**：人物、情节、对话
- **寓意层**：故事背后的哲学概念
- **修辞层**：重言（托古人言）、卮言（随物变化之言）

### 1.2 标记体系

概念标记使用 `〖概念名〗` 格式：
```json
{
  "concept": "逍遥",
  "marker": "〖逍遥〗"
}
```

### 1.3 关系标记

| 类型 | 标记 | 说明 |
|------|------|------|
| 体用 | → | A是B之体，B是A之用 |
| 同一 | = | A＝B |
| 工夫 | => | 通过A达到B |
| 对反 | ↔ | A与B对立又统一 |
| 寓言 | → | 故事映射概念 |

## 二、寓言事件标注格式

```json
{
  "event_id": "Z001",
  "chapter": "逍遥游",
  "title": "鲲鹏之变",
  "source_text": "北冥有鱼，其名为鲲。鲲之大，不知其几千里也。化而为鸟，其名为鹏……",
  "characters": ["鲲", "鹏", "蜩", "学鸠"],
  "summary": "鲲鱼化鹏鸟，扶摇直上九万里，蜩与学鸠嘲笑大鹏之远举。庄子以此展示小大之辨与逍遥之真谛。",
  "key_concepts": [
    {"concept": "逍遥", "marker": "〖逍遥〗"},
    {"concept": "化", "marker": "〖化〗"},
    {"concept": "小大之辨", "marker": "〖小大〗"}
  ],
  "interpretation": "大鹏虽大仍有所待（待风），非真逍遥；蜩鸠之小亦有其适。逍遥不在大小，在无待。",
  "concept_relations": [
    {"source": "〖化〗", "target": "〖逍遥〗", "type": "工夫关系", "description": "鲲化为鹏是突破局限迈向逍遥的象征"},
    {"source": "小", "target": "大", "type": "对反关系", "description": "小大虽殊，各有所待——庄子破除二元对立"}
  ]
}
```

## 四、处理流程

1. **读原文** → 理解寓言和哲学内核
2. **提取寓言事件** → 按寓言为单位分割
3. **标注概念** → 标记关键概念及其关系
4. **解读寓意** → 提供哲学诠释
5. **连接关系** → 构建概念网络
6. **整合审查** → 统一检查后合并
